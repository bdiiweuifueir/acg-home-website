# ACG-Home 功能扩展计划方案 (Feature Expansion Plan)

基于对开源项目 `ChengCheng0v0/ACG-Home` 的分析以及当前项目的重构状态，为您量身定制以下功能扩展方案。本方案旨在增强网站的“二次元”属性，同时保持代码的现代化和高性能。

## 1. 全局背景音乐播放器 (Music Player)
**现状**: 项目中存在 `src/meting.js`，但未正确集成，且依赖全局变量 `window.APlayer`。
**目标**: 集成 APlayer + MetingJS，实现基于歌单 ID 的自动播放。
**实施方案**:
*   **依赖安装**: 使用 NPM 安装 `aplayer` 和 `meting` (或其现代替代品)。
*   **配置化**: 在 `config.json` 中新增 `music` 字段，允许用户配置歌单 ID (网易云/QQ音乐) 和播放模式。
*   **代码实现**: 在 `src/index.js` 中动态加载播放器，确保不阻塞页面首屏渲染。

## 2. Live2D 看板娘 (Live2D Widget)
**现状**: 无。这是二次元网站的标配。
**目标**: 添加一个可交互的 Live2D 人物模型（如 Pio 或 Tia）。
**实施方案**:
*   **技术选型**: 使用 `live2d-widget` 或更轻量级的 `fast-live2d`。
*   **资源管理**: 将模型资源放入 `public/live2d` 目录，或使用 CDN 加载。
*   **配置化**: 在 `config.json` 中新增 `live2d` 字段，控制显示/隐藏及模型选择。

## 3. 动态背景特效 (Dynamic Background)
**现状**: 仅支持静态图片背景。
**目标**: 添加轻量级的动态背景（如线条吸附效果 Canvas-Nest 或 樱花飘落）。
**实施方案**:
*   **技术选型**: `canvas-nest.js` (经典线条) 或 `sakura.js` (樱花)。
*   **性能优化**: 确保特效在低性能设备（如手机）上自动降级或关闭。
*   **配置化**: 在 `config.json` 中控制特效开关。

## 4. 评论系统 (Comment System)
**现状**: 纯静态展示，无互动。
**目标**: 添加评论留言板功能。
**实施方案**:
*   **技术选型**: `Waline` (基于 Vercel/LeanCloud，免费且强大) 或 `Giscus` (基于 GitHub Discussions)。
*   **集成方式**: 在 `index.html` 底部或侧边栏通过组件形式挂载。

---

## 推荐实施优先级

1.  **音乐播放器**: 提升氛围感的核心，且代码已有基础，容易落地。
2.  **Live2D**: 视觉冲击力强，但需注意资源体积。
3.  **动态背景**: 锦上添花，需注意性能。
4.  **评论系统**: 视需求而定，若为纯展示站可暂缓。
